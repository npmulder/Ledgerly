# Module: invoicing

**Package:** `internal/invoicing` · **Schema:** `invoicing` · **Depends on:** moneyfx (rate locks, ToGBP), ledger (postings), jurisdiction (VAT rate, reverse-charge wording), identity (company profile for PDF)

## Responsibility

Clients, invoice lifecycle (draft → sent → paid/overdue), numbering (`INV-YYYY-NN`), PDF rendering, reminders. Screens: 02 Invoices list, 03 Invoice editor, 04 Invoice PDF.

## Domain model

- **Client** — `{id, name, address, vatNumber?, defaultCurrency, terms (e.g. Net 14/Net 30), vatTreatment (domestic | reverse-charge-eu-b2b), retainer?/dayRate?}`. Sample: Contoso GmbH (EUR, €4,500/mo retainer, Net 14, reverse charge), Fabrikam Ltd (GBP, £600/day, Net 30).
- **Invoice** — `{id, number, clientID, status, issueDate, dueDate, currency (defaults from client), lockID?, lines[], vatTreatment, settlement?}`.
  - **Statuses**: `DRAFT` (gray) → `SENT` (teal) → `PAID` (navy); `OVERDUE` (red, badge shows days e.g. `OVERDUE 9D`) is **derived** (status=SENT ∧ dueDate < today), materialized by a daily sweep for filter counts.
  - **Numbering**: `INV-YYYY-NN`, per-year sequence, assigned at first send (drafts unnumbered), gap-free via sequence table row lock.
- **Line** — `{description, qty, unitPrice Money}`. Totals: subtotal, VAT (reverse charge → €0.00 + wording from jurisdiction), total, plus ≈GBP note at locked rate.

## Lifecycle & key flows

```
draft ──send──► sent ──settle (from banking match)──► paid
  ▲ autosave                 └─ dueDate passes ──► overdue (derived)
```

- **Send**: assign number → `moneyfx.Lock(EUR→GBP, issueDate)` → post to ledger (Dr Trade debtors / Cr Sales, native + GBP at locked rate) → publish `invoicing.InvoiceSent`. Editor shows locked rate read-only (source ECB + 🔒).
- **Settle**: banking confirms a match → `MarkSettled(invoiceID, txnRef, date, amount)` → status PAID, publish `invoicing.InvoiceSettled` (moneyfx posts realised FX; ledger gets Dr Cash / Cr Debtors from banking's reconciliation transaction).
- **Reminder**: advisor strip CTA "Send reminder" → SMTP email with PDF attached. v1: manually triggered only.

## PDF (screen 04)

React print route rendered via chromedp. A4 (794px): logo + From (IoM address, company number, from identity) / Bill to (client + VAT no.) / Terms columns; line table with navy double rule; totals with highlighted "Total due"; reverse-charge legal note (wording from jurisdiction, VAT €0.00); SEPA bank details footer; "Generated by Ledgerly". Rendered PDFs stored (path on invoice) so a sent invoice's document never changes retroactively.

## Public API (Go)

```go
type Invoicing interface {
    // commands
    CreateDraft / UpdateDraft / Send / MarkSettled / SendReminder
    // queries
    Invoice(id) · List(filter{status, search}) · OverdueInvoices() // advisor fact
    Clients() · SaveClient()
    Totals(filter) // list-footer totals
}
```

## Events

Publishes: `InvoiceSent`, `InvoiceSettled`, `InvoiceOverdue` (sweep). Consumes: none (settlement arrives via API call from banking within its transaction).

## Data (schema `invoicing`)

`clients`, `invoices`, `invoice_lines`, `invoice_numbering`, `rendered_docs`.

## UI notes (screens 02–03)

Status filter pills with counts; search; table columns number/client/issued/amount/locked rate/GBP/status; overdue rows tinted `#fdf5f6`; footer totals; advisor strip.
